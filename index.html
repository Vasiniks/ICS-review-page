Top 5 Study Strategies for Paper Exams
  • 1. Write Code on Paper: Since the exam is paper-only, practice coding by hand. Pay attention to colons (:) after conditions/functions, parentheses, and indentation spaces.
  • 2. Trace Variables step-by-step: Maintain a margin table tracing variable updates. Never try to hold loops entirely in your head!
  • 3. Learn Callback Differences: Be absolutely clear on widget commands: Scale and OptionMenu pass an argument to their callback functions. Button and Spinbox do not.
  • 4. Understand slicing ranges: In s[start:stop], the start index is inclusive while the stop index is exclusive (stops right before the stop index).
  • 5. Draw Widget Layout Grids: Sketch grid placements on paper. Count row spans and column spans before translating layouts into code.

Python Part 1 (py1)

Fundamental data types, mathematical expressions, and libraries.

Variable Data Types

Python stores data in distinct memory types. The main four structures are:

  • int: Whole values (e.g. 10, -5).
  • float: Decimal fractional numbers (e.g. 3.14, -0.5).
  • str: String text wrapped in quotes (e.g. "hello").
  • bool: Logical states (True or False - note the capitalized first letter).
Arithmetic & Modulo Operations

Operator math is evaluated by BEDMAS precedence. Modulo (%) and floor division (//) are highly tested on paper exams:

Operator Function Example Result
** Power / Exponentiation 5 ** 2 25
/ Float Division (returns decimals) 5 / 2 2.5
// Integer / Floor Division (rounds down) 5 // 2 2
% Modulo (remainder after division) 5 % 2 1
Formatted Outputs (f-strings)

Use custom spacing inside string outputs:

  • Rounding float decimals: f"{price:.2f}" rounds value to exactly 2 decimal places.
  • Centering text: f"{word:^30}" centers string text within a 30-character gap block.
  • Alignment: f"{word:>20}" aligns text to the right side; f"{word:<20}" aligns left.
The 4 Key Random Functions

Import library via import random as line 1. These four functions are highly tested:

1. random.randint(a, b)
Returns a random integer. Both limits a and b are INCLUSIVE.

2. random.randrange(start, stop, step)
Returns a random integer from range. The stop parameter is EXCLUSIVE.

3. random.uniform(a, b)
Returns a random floating decimal number between a and b (both inclusive).

4. random.choice(sequence)
Selects and returns a single random element from a list, tuple, or string.

Python Part 2 (py2)

Loop counts, sequence slicing, string functions, and mutability behavior.

For Loops & Range Limits

Loops execute statements. For loops count using range settings: range(start, stop, step) (excluding stop):

# Counts down from 1000 to 800 in steps of -10
for i in range(1000, 799, -10):
    print(i)
Slicing Sequences: s[start:stop:step]

Extract sub-parts of strings/lists. The start index is inclusive, while the stop index is exclusive:

phrase = "quick brown"
print(phrase[6:9])     # Output: "bro"
print(phrase[:5])      # Output: "quick"
print(phrase[::2])     # Output: "qi bwn"
print(phrase[-2:])     # Output: "wn" (Negative indices count from right end)
Mutability Concept (Exam Trick Warning)

Strings are Immutable: String methods do not modify the original variable. They return a new value. You must assign it back.

word = "python"
word.upper()        # Trick! Does nothing to the actual variable 'word'
print(word)         # Output: "python"

word = word.upper() # Reassigning stores it
print(word)         # Output: "PYTHON"

Lists are Mutable: Lists are updated in place. However, `.sort()` returns None. Never assign a list to its own sort output!

my_list = ["x", "a", "q"]
my_list.sort()      # Sorts list in-place!
print(my_list)      # Output: ["a", "q", "x"]

# DANGER EXAM TRICK:
my_list = my_list.sort() # my_list is now None!
print(my_list)      # Output: None

Tkinter GUI

Complete widget options, states, configurations, and variables.

variable vs textvariable

This is a major source of syntax errors on written tests:

  • textvariable: Used when a widget needs to bind to a StringVar or IntVar to dynamically display or read text. Used by: Label, Entry, Button, and Spinbox.
  • variable: Used for widgets that track states, values, or sliders directly rather than arbitrary string inputs. Used by: Checkbutton, Scale, and Radiobutton.
Widget Configuration & State Options

Tkinter widgets are initialized with settings, but they can also be dynamically modified using .config() or subscript notation:

# Example of configuring widget properties on the fly:
submit_button.config(text="Sending...", state=DISABLED)

# Alternate dictionary syntax:
submit_button["state"] = DISABLED

Common States:

  • NORMAL (or "normal"): Widget is active, clickable, and responsive (default).
  • DISABLED (or "disabled"): Widget is grayed out, unclickable, and ignores user interaction. Useful for disabling buttons after submission.
  • "readonly": Specific to widgets like Entry and Spinbox. The user can see the text but cannot modify it directly.
Detailed Widget Explanations & Initialization

1. Label: Shows static text or updates via textvariable.

# text: static string
# font: tuple of (font_name, size, weight)
my_label = Label(mainframe, text="Static Text", font=("Arial", 12, "bold"))

# textvariable: links to StringVar to auto-update
name_var = StringVar()
dynamic_label = Label(mainframe, textvariable=name_var)

2. Entry: Text input field for user entry.

input_var = StringVar()
# width: size of the input box in character count
# show: replaces characters with symbols (e.g. show="*" for password entry)
password_entry = Entry(mainframe, textvariable=input_var, width=15, show="*")

3. Button: Clickable element to run code.

# command: binds callback function (do not add parentheses!)
click_btn = Button(mainframe, text="Submit", command=my_function)

4. OptionMenu: Dropdown box selector.

Unpacking List Option Syntax: Options must be passed as individual positional arguments. We prefix a standard python list with an asterisk * to unpack it.

choices = ["First", "Second", "Third"]
choice_var = StringVar()
choice_var.set(choices[0])

# Category_menu = OptionMenu(parent, variable, *values, command=callback)
# Note: command callback function MUST accept 1 parameter (the selected string)
category_menu = OptionMenu(mainframe, choice_var, *choices, command=my_callback)

5. Spinbox: Scroll selector box with adjust arrows.

spin_var = StringVar()
# values: tuple/list of choices to navigate (like OptionMenu)
# from_/to: numeric boundaries instead of explicit lists
# command: runs when arrow is clicked (takes 0 arguments!)
number_spin = Spinbox(mainframe, textvariable=spin_var, from_=1, to=10, command=my_callback)

6. Scale: Sliding scale selector.

slider_var = IntVar()
# from_: starting limit value (note trailing underscore!)
# to: maximum limit value
# resolution: step increments (e.g. resolution=5 counts 5, 10, 15...)
# command: runs when slider shifts (MUST accept 1 parameter!)
tip_scale = Scale(mainframe, from_=0, to=30, resolution=5,
                  variable=slider_var, orient=HORIZONTAL, command=on_shift)

7. Checkbutton: Checkbox element.

check_var = IntVar() # stores 1 if checked, 0 if unchecked
# onvalue/offvalue: custom values (e.g. onvalue=0.13 to store decimal tax float)
tax_check = Checkbutton(mainframe, text="Tax?", variable=check_var, onvalue=1, offvalue=0)

8. Listbox: Scroll selection list box.

todo_var = StringVar()
todo_var.set(["Work", "Eat", "Sleep"])
# height: visible row size count
todo_listbox = Listbox(mainframe, listvariable=todo_var, height=3)

# Reading value trace:
# index = todo_listbox.curselection()[0] # returns index of chosen item

9. PhotoImage: Load and display image graphics.

# Load graphic (supports PNG and GIF in python 3)
flag_img = PhotoImage(file="flag.png")
# Must display it on a Label or Button
img_label = Label(mainframe, image=flag_img)

# CRITICAL gotcha: you must keep a reference to prevent garbage collection!
# img_label.image = flag_img

Challenge 2: Odd/Even String Filter Loop

Write a function filter_even_chars(s) that takes a string and returns a new string containing only characters at even indices.

⚠️ Click to reveal answer and trace steps
Answer Code: 
def filter_even_chars(s):
    result = ""
    for i in range(len(s)):
        if i % 2 == 0:
            result += s[i]
    return result

# Alternative Slicing Answer:
def filter_even_chars_slice(s):
    return s[::2]

Challenge 3: Scale Callback Payout

Write a callback function calculate_payout(val) bound to a Tkinter Scale widget. The base wage is stored in wage_var (StringVar). The hours worked is bound to hours_var (IntVar). Overtime (hours > 40) is paid at 1.5x. Display results in salary_var (StringVar) formatted as "$XX.XX".

⚠️ Click to reveal answer and trace steps
Answer Code: 
def calculate_payout(val):
    rate = float(wage_var.get())
    hours = hours_var.get()
    
    if hours <= 40:
        pay = rate * hours
    else:
        overtime = hours - 40
        pay = (rate * 40) + (rate * 1.5 * overtime)
        
    salary_var.set(f"${pay:.2f}")

Challenge 4: List Average Calculation

Write a function list_average(some_list) that calculates and returns the average of all numbers in a list using the built-in functions sum() and len().

⚠️ Click to reveal answer and trace steps
Answer Code: 
def list_average(some_list):
    if len(some_list) == 0:
        return 0
    return sum(some_list) / len(some_list)

Challenge 8: In-Place Sort Traps

Trace the output of this Python list manipulation program:

names = ["Kilo", "Echo", "Sierra"]
names.sort()
print(names)

names = names.append("Alpha")
print(names)
⚠️ Click to reveal answer and trace steps
Printed Output: 
['Echo', 'Kilo', 'Sierra']
None
Trace Explanation:
  • names.sort() sorts in-place. Result is ['Echo', 'Kilo', 'Sierra'].
  • names.append("Alpha") appends to the list in-place and returns None. By assigning names = names.append("Alpha"), we store None inside the names variable!

Challenge 9: Loop Accumulation Sum

Write a function sum_multiples(n) that uses a loop to sum all numbers from 1 to n (inclusive) that are divisible by 3 or 5.

⚠️ Click to reveal answer and trace steps
Answer Code: 
def sum_multiples(n):
    total = 0
    for i in range(1, n + 1):
        if i % 3 == 0 or i % 5 == 0:
            total += i
    return total
